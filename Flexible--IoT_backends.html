Flexible open source-based IoT backends in the cloud

With solutions ranging from stream ingestion to big data, machine learning, and visualization, Microsoft Azure empowers open source developers to realize IoT needs in the cloud.

Solution brief

The Azure end-to-end solution for big data enables the collection of any type of data from any source. Azure processes the data and visualizes results with meaningful insights. Azure also provides a complete suite of services for connection of IoT devices, data analysis, machine learning, and data visualization. This comprehensive solution showcases how Azure can be leveraged to build robust platforms that meet your big data requirements.

Business data is growing rapidly and will continue to increase in the future. According to IDC, a leading market research company, the digital universe will grow 40 percent each year through the next decade. This data growth not only includes data from additional users or enterprises working online, but also data coming from smart devices connected to the Internet.

Business processes depend on customer requirements, competition, and innovation. And better insight into business data plays an important role in identifying new trends, strategies, requirements, and competitors. Big data analysis is becoming an essential tool for business decision makers (BDMs). Organizations are looking for quick and reliable ways to process large amounts of data coming from disparate sources, including business applications, mobile platforms, and Internet of Things (IoT) devices. Microsoft Azure enables you to handle any amount of data, scaling from terabytes to petabytes on demand while deploying multiple scenarios. Azure offers robust big data solutions with a range of enterprise-grade services for data collection, storage, analysis, and presentation for accurate insights into big data.

This scenario focuses on how Azure provides end-to-end support for both the front-end and back-end services driven by Linux and open source solutions.

Network Tier

The Azure network tier includes high-performance network services for applications running in development and test environments on Azure. These services provide different functionalities, including secure access, name resolution, content delivery, and load balancing. The following services can be used with your applications on Azure.

Azure ExpressRoute

This service establishes a connection between your on-premises datacenter and Azure, without going through public Internet.

To configure ExpressRoute:

  • Gather information about connectivity providers and locations
  • Create an ExpressRoute circuit
  • Configure routing (circuit peering)
  • Link a VNet to an ExpressRoute circuit

For more information: ExpressRoute circuits and routing domains

VPN Point-to-Site or Site-to-Site

Create a VPN connection over IPsec (IKE v1 and IKE v2). This type of connection requires a VPN device or routing and remote access service (RAS).

To configure Azure VPN Gateway:

  • From the Azure portal, create a VPN gateway
  • Gather information for your VPN device configuration
  • Configure your VPN device
  • Verify your local network ranges and VPN gateway IP address

For more information: Configuring a VPN gateway

Data Collection Tier

For any big data solution, it is important to know what types of data sources and devices are supported for data collection. Organizations want a solution that can quickly gather huge amounts of data with consistency. Azure provides options for collecting and ingesting data from different sources with Azure Event Hubs, Azure Service Bus, and other storage services.

Azure Event Hubs

Organizations need to gather millions of events from multiple applications and IoT devices. Azure Event Hubs is a highly scalable publish-subscribe event ingestor that can intake millions of events per second, enabling you to process and analyze massive amounts of data. Once collected in Event Hubs, you can transform and store data with any real-time analytics provider or with batching/storage adapters.

For more information: Azure Event Hubs

To create and configure Event Hubs to run applications:

  • Access your Azure subscription
  • Create an Event Hub
  • Send messages to Event Hubs
  • Receive messages with EventProcessorHost/Apache Storm
  • Run the applications

For more information: Get started with Event Hubs

Azure Service Bus

Azure Service Bus is a managed messaging system that connects with business applications, services, and devices in the cloud, on-premises, or in hybrid scenarios. You can even use Service Bus to connect enterprise appliances, sensors, and other devices—like tablets or phones―to a central application or to each other.

For more information: Azure Service Bus

To configure Service Bus queues:

  • Access your Azure subscription
  • Create a service namespace
  • Obtain the default management credentials for the namespace
  • Configure the application to use Service Bus
  • Create a queue
  • Send a message to the queue
  • Receive a message from the queue

Note: Service Bus allows you to create and configure queues for different programming languages, including Java, PHP, Node.js, and Python.

For more information: How to use Service Bus queues

Storage Tier

Data collection services gather structured, semi-structured, and unstructured business data through different types of storage options. Data storage services should be scalable enough to store large amounts of business data, and Azure provides several storage options for any type of data.

Azure Blob storage

Azure Blob storage can store huge amounts of unstructured data—even millions of documents, logs, images, videos, audio files, and more. Blob storage also offers several benefits over the traditional Hadoop Distributed File System (HDFS) for big data scenarios, including geo-replication, lower storage costs, easier scalability, and data sharing outside of the Hadoop cluster.

For more information: Azure Blob storage

To configure Azure Blob storage:

  • Get the Azure Client Library
  • Configure your application to access the Azure Blob service
  • Set up an Azure storage connection
  • Create a container
  • Upload a blob to a container
  • List the blob in the container

Note: Azure Blob storage allows you to create and configure blobs for different programming languages, including Java, PHP, Node.js, and Python.

For more information: How to use Azure Blob storage from PHP

Azure File storage

Azure File storage offers fully managed file shares, allowing you to share files across on-premises and Azure cloud platforms. Availability, durability, scalability, and geo-redundancy are built into the Azure storage platform, allowing you to confidently store server data and backups in the cloud. Using SMB encryption, you can also securely mount Azure File storage from anywhere.

For more information: Azure File storage

To configure Azure File storage:

  • Set up your application to use Azure File storage
  • Set up an Azure storage connection string
  • Connect to an Azure storage account
  • Create a file share
  • Upload a file
  • Create a directory
  • List files and directories in the share

For more information: How to use File storage from Java

Azure Table storage

Azure Table storage is perfect for flexible data sets—web application user data, address books, device information, and other metadata—and lets you build cloud applications without locking down the data model to particular schemas. Azure Table storage can store petabytes of semi-structured data while keeping costs down, and it lets you scale up without having to manually shard your data set.

For more information: Azure Table storage

To configure Azure Table storage:

  • Get the Azure storage client library
  • Configure your application to access the table service
  • Set up an Azure storage connection
  • Create a table
  • Add an entry to a table
  • Retrieve single or multiple entities
  • Perform batch table operations

Note: Table storage allows you to create and configure tables for different programming languages, including Java, PHP, C++, Node.js, and Python.

For more information: How to use Azure Table storage from PHP

Azure Data Lake Store

Azure Data Lake Store provides an enterprise-class scalable data repository for big data analytics workloads. Data Lake allows you to store data from any source, including business applications, smart phones, IoT devices, and sensors. Data Lake Store provides the scalability and flexibility to store data of any size and type with quick ingestion.

For more information: Azure Data Lake Store overview

Azure gives you multiple options to configure Data Lake Store using Azure portal, PowerShell, .NET SDK, and Azure CLI. To configure Data Lake Store using the Azure portal:

  • Enable your Azure subscription for Data Lake Store public preview
  • Create an Azure Data Lake Store account
  • Create folders in the account
  • Upload data to the account
  • Secure your data

For more information: Azure Data Lake Store

Azure DocumentDB

Applications produce, consume, and respond quickly to very large volumes of data. Modern applications are built using schema-free NoSQL document databases as simple, fast, scalable solutions to store and process data. Azure DocumentDB is a true schema-free NoSQL document database service designed for modern mobile and web applications. DocumentDB delivers consistently fast reads and writes, schema flexibility, and the ability to scale a database up or down on demand.

DocumentDB enables complex ad hoc queries, supports well-defined consistency levels, and offers JavaScript-integrated, multi-document transaction processing using the familiar programming model of stored procedures, triggers, and UDFs. DocumentDB natively supports JSON documents, enabling easy iteration of application schema. It embraces the ubiquity of JSON and JavaScript, eliminating mismatch between defined objects and database schema.

For more information: Azure DocumentDB

To configure DocumentDB:

  • Create a DocumentDB account
  • Set up your Node.js application
  • Set your application’s configurations
  • Connect to your DocumentDB account
  • Create a node database and collection
  • Create a document
  • Query DocumentDB resources
  • Delete the node database
  • Put it together and run your Node.js application

Note: DocumentDB exposes resources through a REST API that can be called by any language capable of making HTTP/HTTPS requests.

To create a DocumentDB Account with the specified name and a default consistency level of session:
Azure Resource Manager template

MySQL on Azure Virtual Machines

To store your application and service data for big data requirements, you can also use MySQL on Azure Virtual Machines. MySQL is an open source database for web and cloud-based applications that allows you to integrate with numerous big data platforms. Azure provides pre-configured virtual machine images for MySQL that can help you quickly build and configure your database on Azure.

To configure MySQL on Azure Virtual Machines, you must first provision a new virtual machine for MySQL. There are two ways to do this:

  • Use a pre-configured image for MySQL from the Azure Marketplace
  • Provision a base Linux virtual machine, and then manually deploy and configure MySQL to store your gathered data

Data Pipeline Tier

After storing your data within the storage tier, you will need to move the data to Hadoop clusters for data analytics. A data pipelining solution logically groups data into multiple activities that need to be orchestrated and automated, and then these activities are placed into a single pipeline for Hadoop clusters. Azure Data Factory (ADF) is used to create highly available dataflow pipelines for scenarios across different industries for their analytics pipeline needs.

Azure Data Factory

Azure Data Factory offers integration that orchestrates and automates the movement and transformation of data. It provides simplified ingestion of data from multiple cloud or on-premises sources and a wide range of processing services. ADF uses data pipelines to transform raw data into shaped data that is ready for consumption by BI tools or applications. ADF allows you to visualize, monitor, and manage your entire network of data pipelines at a glance to identify issues and take action.

For more information: Azure Data Factory

To create a data factory and pipeline:

  • Create the data factory
  • Create linked services and data sets
    • Create an Azure Storage linked service
    • Create an Azure HDInsight linked service
    • Create the output dataset
  • Create your first pipeline

For more information: Get started with Azure Data Factory

Apache Hadoop Tier

Azure HDInsight deploys and provisions Apache Hadoop clusters in the cloud, providing a software framework designed to manage, analyze, and report on big data with high reliability and availability. HDInsight uses the Hortonworks Data Platform (HDP) Hadoop distribution. Hadoop often refers to the entire Hadoop ecosystem of components, which includes Storm and HBase clusters, as well as other technologies under the Hadoop umbrella. HDInsight provides cluster configurations for Hadoop, HBase, and Storm. You can also customize clusters with script actions. HDInsight uses either Linux or Windows as the underlying OS.

  • HDInsight on Linux: Use this if you are familiar with Linux or UNIX, migrating from an existing Linux-based Hadoop solution, or want easy integration with the Hadoop ecosystem components built for Linux.
  • HDInsight on Windows: Use this if you are familiar with Windows, migrating from an existing Windows-based Hadoop solution, or want to use .NET or other Windows-only technologies on the cluster.

For more information: Introduction to Hadoop components in HDInsight

HDInsight

Machine Learning and Visualization Tier

After running Hadoop jobs, you can either forward your data to machine learning systems for behavioral or usage pattern monitoring, or you can send your data back to the storage tier. Machine learning solutions provide accurate predictive analyses based on application and device behavior to make your application and devices smarter. Azure Machine Learning provides these cloud-based predictive analytics, and it allows you to integrate with visualization tools such as Power BI for data visualization and easier decision-making.

Azure Machine Learning

Azure Machine Learning is a comprehensive predictive analytics service that helps you analyze data and quickly create predictive analyses. It uses computers to run predictive models that analyze existing data and learn in order to forecast future behaviors, outcomes, and trends. These predictions can help make your applications and devices smarter.

For more information: Azure Machine Learning

To configure Azure Machine Learning:

  • Go to the Azure Machine Learning site and click Get started (you can sign in with your Microsoft account or choose Guest Access)
  • Create a model
    • Get data
    • Pre-process the data
    • Define features
  • Train the model
    • Choose and apply a learning algorithm
  • Score and test the model
    • Perform predictive analysis

For more information: Azure Machine Learning

Power BI

Power BI is a cloud-based service that allows you to directly connect with your business data, quickly create dashboards and reports, and share reports throughout your organization. Power BI seamlessly works with different Azure services to provide effective reporting with real-time insights into your business. Power BI provides a wide range of visualization types, including charts, card tiles, maps, and matrixes.

For more information: Power BI

To analyze and visualize data with Power BI:

  • Log on to Power BI
  • Add databases
  • Connect with Spark on HDInsight (once a Spark cluster is connected, Power BI imports data from the cluster and displays it in a new dashboard)
  • Build visualizations to show desired results
  • Save your visualizations to be available from your dashboard in the future (Spark clusters on HDInsight connect to Power BI directly, so Power BI always has the most updated data and does not require data set refreshes)

Other BI Tools

Apart from Power BI, HDInsight supports various third-party tools to get actionable and interactive visualizations, including Tableau, Qlik, and SAP Lumira. These tools allow you to visualize results from petabytes of data.

For more information: Apache Spark

Management Tier

Microsoft provides a variety of tools for managing Azure resources. This section summarizes the different management tools for Azure.

Azure management tools

  • Azure portal: Lets you build, manage, and monitor everything from simple web apps to complex cloud apps from a single, unified console.
  • Azure CLI (command-line interface): Offers a set of open source, shell-based commands for creating and managing resources in Azure.
  • Azure Resource Manager (ARM): Enables you to work with the resources in your solution as a group. You can deploy, update or delete all of the resources for your solution in a single, coordinated operation. ARM provides a wide range of community-based templates built by ARM community members, who continuously contribute to new templates. These templates can be used for deployments or different environments such as testing, staging, and production.
  • Azure API Management: Provides a REST API for performing operations on selected entities, including users, groups, products, and subscriptions.
  • Language-specific SDKs (.NET, Java, Node.js, PHP, and Python): Helps manage your Linux environment.
