Azure Load Balancer

Azure Load Balancer delivers high availability and network performance to your applications with application instance health probe and automatically scalability with increasing traffic demand of applications to avoid bottle-necks. Once user requests your application hosted on Azure platform, traffic routes through scalable load balancer to distribute incoming traffic among healthy application instances in virtual machines. You can use Azure Load Balancer for TCP/UDP-based protocols such as HTTP, HTTPS, SMTP as well as protocols used for real-time voice, video messaging applications.

For more information about Azure Load Balancer, visit here.

  • Add virtual machines to the Load Balancer Set
  • Configure Distribution mode for Load Balancer
  • Check load balancer health status

Web Tier

Application front-end such as web and mobile services is critical part of any application that provides interface to users to interact. Azure provides platform to let you rapidly provision web and mobile applications on Azure Virtual Machine. Azure Virtual Machine supports Linux operating systems to deploy your open source application. You can migrate existing application server virtual machine images (vhd) from on-premises to Azure or deploy application on newly provisioned Linux virtual machine.

For high availability, two or more Linux virtual machines can be used under web tier in the same availability set. An availability set helps keep your virtual machines available during downtime providing redundancy needed to maintain availability of the applications or services that your virtual machine runs. Following are the high level steps to configure your Web Tier on Azure:

  • Go to your Azure Subscription
  • Choose Azure Virtual Machine image created previously of your on-premises Web Server to provision new VM
  • Create two virtual machines within a single availability set to make your Web Tier highly available
  • Connect your virtual machines with SSH

Cloud Burst on-premises application traffic

This hybrid solution can provide cloud bursting for your lob application to the Azure cloud infrastructure to handle additional user demand in the peak time. To enable cloud bursting on Azure based application infrastructure, you need to configure thresholds on on-premises traffic manager to route web traffic to Azure hosted application servers. Whenever your application demand increases and breach threshold on on-premises traffic manager, it can route additional user traffic to your business application available on the Azure infrastructure.

After successfully configuring Network Tier, Identity Tier, Web Tier, App Tier and Database Tier, you are almost ready with your application infrastructure on Azure. Now to enable cloud-burst scenario for your application follow below steps:

  • Define limit for user traffic for your web application to be handled by your on-premises infrastructure
  • Configure thresholds for limiting user traffic for your on-premises application and define condition to route rest of the traffic to the application resides in Azure infrastructure

You can provision preconfigured database virtual machine images or manually deploy database servers on Linux Azure Virtual Machines. Once database servers are provisioned, you can attach data vhd with backup of on-premises database of application. Different approaches available to deploy database servers with high availability. A multi-master cluster or traditional master slave cluster can be created to deploy robust, scalable, and reliable database tier for your application on Azure Virtual Machines.

Internet

Organizations are more concern about the security aspects of the cloud. Whether your application is private business application or public website or web app, Azure provides security option to provide robust security options. You can secure end-to-end connectivity between your datacenters and Azure cloud infrastructure for your business application. This enables you to build truly hybrid applications in Azure. Azure provides two ways to secure your connection either using directly connection to Azure from your existing WAN network or virtual private network (VPN) over public internet. Azure Virtual Private Network secures your connection with site-to-site VPN using IPsec. Azure ExpressRoute offers secure and faster direct connection to the Azure infrastructure without going over public internet. With ExpressRoute, you can establish connections to Azure at an ExpressRoute location, such as an Exchange provider facility or directly connect to Azure from your existing WAN network, such as a multi-protocol label switching (MPLS) VPN, provided by a network service provider.

Management

To manage Azure resources, Microsoft provides different tools for different type of requirements and skillsets. Following are different management tools for Azure:

  • Azure Portal– Build, manage and monitor everything from simple web apps to complex cloud applications in a single, unified console.
  • Azure CLI– The Azure CLI provides a set of open-source shell-based commands for creating and managing resources in Microsoft Azure.
  • Rest APIs– Azure API Management provides a REST API for performing operations on selected entities, such as users, groups, products, and subscriptions.
  • SDKs– Azure provides multiple language-specific SDKs to manage your Linux environment. These languages can be .NET, Java, Node.js, PHP and Python.
