Cloud-powered development and testing with open source workflow integration

Organizations are searching for innovative solutions to maintain infrastructure for development and testing. Development and test environments should be scalable enough to match the application configuration in the production environment to test performance and functionality with a similar load. Moving forward from traditional application development and deployment, the industry is adopting cloud-based development for an easier packaging and deployment approach.

Linux containers are the next-level approach over traditional virtual machine-based deployment. Containers can effectively isolate data and computing on shared resources and enable virtual machine capabilities with kernel containment rather than simulating and running a separate kernel with each virtual machine. Docker-based containers can help you rapidly provision application deployment for your development and test environment, and securely isolate these applications from other applications deployed on the same Docker host. With a distributed microservices model using Docker containers, your application can be quickly changed as needed to better serve the end user.

Microsoft Azure provides a platform to build cloud-based applications, without having to manage a complex and expensive infrastructure deployment. You can create lightweight, private platform-as-a-service (PaaS) environments for development and testing, and also automate packaging of applications with Docker-based containers. Azure provides additional capabilities to simplify deployment of Docker Engine and manage remote communication with the virtual machine using Docker Virtual Machine Extension. Azure allows you to build your development and test environment with low compute capacity and provide flexibility to scale up to make a production-identical environment. With Azure, you pay only for what you use and eliminate the underutilization of resources in traditional development and test environments.

Network Tier

The Azure network tier includes high-performance network services for applications running in development and test environments on Azure. These services provide different functionalities, including secure access, name resolution, content delivery, and load balancing. The following services can be used with your applications on Azure.

Azure Express Route

This service establishes a connection between your on-premises datacenter and Azure, without going through public Internet.

To configure ExpressRoute:

  • Gather information about connectivity providers and locations
  • Create an ExpressRoute circuit
  • Configure routing (circuit peering)
  • Link a VNet to an ExpressRoute circuit

For more information: ExpressRoute circuits and routing domains

VPN Point-to-Site or Site-to-Site

Create a VPN connection over IPsec (IKE v1 and IKE v2). This type of connection requires a VPN device or routing and remote access service (RAS).

To configure Azure VPN Gateway:

  • From the Azure portal, create a VPN gateway
  • Gather information for your VPN device configuration
  • Configure your VPN device
  • Verify your local network ranges and VPN gateway IP address

For more information: Configuring a VPN gateway

Azure DNS

The first step to access a web application is to get name resolution from web address to IP address. DNS domains in Azure Domain Name Services (Azure DNS) are hosted on the Azure global network of DNS name servers. To provide both fast performance and high availability for your domain, the DNS query is answered by the closest available DNS server. By hosting your domains in Azure, you can manage your DNS records using the same credentials, APIs, tools, and billing as your other Azure services.

For more information/free trial: Azure DNS preview site

You can configure Azure DNS using Azure command-line interface (CLI) or Windows PowerShell. To configure DNS for your application:

  • Switch CLI to use Azure Resource Manager (ARM) commands and DNS
  • Create DNS zones
  • Create records and record sets
  • Delegate domain to Azure DNS

For more information: Getting started with Azure DNS using CLI

To create a DNS zone within Azure DNS and add record sets: Azure Resource Manager template

Azure Load Balancer

Azure Load Balancer and Azure Traffic Manager deliver a highly available network to test performance of your business application in case of unexpected traffic load. During testing, your application hosted on the Azure platform may receive a large number of user requests. In this case, incoming traffic routes through the scalable Load Balancer or Traffic Manager to distribute it among healthy service instances in cloud services or virtual machines.

Azure Load Balancer can be configured to route Internet and intranet traffic to hosted services. If your development and test workload is hosted across different Azure datacenters, you can use Traffic Manager to distribute user traffic to hosted services within or across datacenters.

For more information: Load Balancer and Traffic Manager

To configure incoming Internet and intranet traffic load balancing to Azure virtual machines using Azure Load Balancer:

  • Create the Load Balancer set
  • Add virtual machines
  • Configure the distribution mode
  • Check the Load Balancer health status

For more information: Load Balancer

To create a Load Balancer with Inbound NAT Rule: Azure Resource Manager template

DocumentDB

Applications produce, consume, and respond quickly to very large volumes of data. These applications evolve rapidly and so does the underlying data schema. Modern applications are built using schema-free NoSQL document databases as simple, fast, and scalable solutions to store and process data. DocumentDB is a true schema-free NoSQL document database service designed for modern mobile and web applications. DocumentDB delivers consistently fast reads and writes, schema flexibility, and the ability to easily scale a database up and down on demand.

DocumentDB enables complex ad hoc queries using a SQL language, supports well-defined consistency levels, and offers JavaScript-language integrated, multi-document transaction processing using the familiar programming model of stored procedures, triggers, and user-defined functions (UDFs). DocumentDB natively supports JSON documents, enabling easy iteration of application schema. It embraces the ubiquity of JSON and JavaScript, eliminating mismatch between applications-defined objects and database schema.

For more information: DocumentDB webpage

To configure Azure DocumentDB to true, schema-free NoSQL document database service for your application:

  • Go to your Azure subscription
  • Create a DocumentDB Account
  • Set up your Node.js application
  • Set your app configurations
  • Connect to a DocumentDB account
  • Create a node database and collection
  • Create a document
  • Query DocumentDB resources
  • Delete the node database
  • Put it together, and run your Node.js application

Note: Azure DocumentDB exposes resources through a REST API that can be called by any language capable of making HTTP/HTTPS requests.

For more information: Introduction to DocumentDB: A NoSQL JSON database

To create a DocumentDB account with the specified name and a default consistency level of session: Azure Resource Manager template

Redis Cache

Azure Redis Cache helps your application become more responsive even as user load increases and leverages the low latency, high-throughput capabilities of the Redis engine. This separate, distributed cache layer allows your data tier to scale independently for more efficient use of compute resources in your application layer.

Redis is an advanced key-value store in which keys can contain data structures such as strings, hashes, lists, sets, and sorted sets. Redis supports a set of atomic operations on these data types. Azure allows you to use Redis Cache with different web development languages and platforms, including .Net, Node.js, Java, Python, PHP, and WordPress.

  • Add and retrieve objects from the cache

Note: With Azure Redis Cache, you can also configure caching services for your application with Node.js, Java, and Python. Configuring different languages on Azure Redis Cache will require different configuration steps.

Azure management tools

Microsoft provides Azure management tools for many types of requirements and skill sets, including:

  • Azure portal: Build, manage, and monitor everything from simple web apps to complex cloud applications in a single, unified console.
  • Azure CLI: Azure CLI provides a set of open source, shell-based commands to create and manage resources in Azure.
  • Azure Resource Manager: Azure Resource Manager (ARM) enables you to work with the resources in your solution as a group. You can deploy, update, or delete all of the resources for your solution in a single, coordinated operation. ARM provides a wide range of community-based templates built by ARM community members. Community members are continuously contributing to new templates. These templates can be used for deployments or different environments such as testing, staging, and production.
  • Rest APIs: Azure API management provides a REST API for performing operations on selected entities, such as users, groups, products, and subscriptions.
  • SDKs: Azure provides multiple language-specific SDKs to manage your Linux environment. These languages can be .NET, Java, Node.js, PHP, Python, and Ruby.

OMS for Linux

Operations Management Suite provides an easy-to-install agent for Linux that allows you to collect Syslog events, performance metrics, and container data for log analytics.

For more information: Operations Manager Engineering Blog

Management tools for Dockers

You can use the following Docker and third-party tools to simplify Docker management:

  • Docker Machines: Docker Machines allows you to automatically create host machines and install Docker Engine on them. You can also configure a Docker client on the host.
  • Docker SWARM: Docker Swarm provides native clustering for multiple Docker hosts. Using a full suite of Docker tools, you can create and access Docker pools.
  • Docker Compose: Docker Compose is a tool to define and run multi-container Docker applications. A Compose file can help configure your applicationâ€™s services.
  • Kubernetes: Kubernetes allows you to manage a cluster of Docker-based Linux containers as a single system to accelerate development and simplify IT operations.
  • Apache Mesos: Mesos supports the launch of tasks that contain Docker images. You can either launch a Docker image as a task, or as an executor.
